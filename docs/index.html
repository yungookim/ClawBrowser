ClawBrowser: A Proactive,
Memory-Persistent AI Browser

Yungoo Kim

February 2026

Abstract

We present ClawBrowser, a lightweight open-source web browser in which an AI agent is integrated at the architectural level rather than appended as an extension or sidebar. Unlike existing AI-augmented browsers that operate reactively and discard context between sessions, ClawBrowser maintains persistent long-term memory, runs proactive background monitoring, and performs nightly self-reflection to continuously refine its understanding of the user. The system supports multiple AI backends, processes voice input locally, stores all data on-device, and ships as a single binary under 10 MB. This paper describes the design philosophy, core architecture, and distinguishing characteristics of the system.

1. Introduction

The modern browser is, for most people, the primary interface to the internet. Over the past two years, several companies have attempted to augment that interface with artificial intelligence. Perplexity released Comet; OpenAI released Atlas. Both follow the same pattern: embed a language model beside the browsing experience and let the user invoke it on demand.

The result is a fundamentally reactive system. The user asks a question; the model answers. The user gives a command; the model executes it. When the tab closes, the context is gone. This interaction model treats the AI as a tool to be picked up and put down—useful, but stateless and passive.

ClawBrowser begins from a different premise: the AI should not sit beside the browser. It should be the browser—or at least, be so deeply woven into its fabric that the distinction between “browsing” and “asking the AI” ceases to exist. The agent sees every page the user sees, remembers what matters across sessions, anticipates needs before they are articulated, and improves its own behavior through nightly self-evaluation.1

This paper describes the architecture and design choices that make this possible.

2. Related Work

Comet (Perplexity, 2025) integrates Perplexity’s search-and-answer engine into a Chromium-based browser shell. The AI is accessible through a sidebar and can answer queries about the current page. It does not maintain memory between sessions, operates only when invoked, and is locked to Perplexity’s proprietary model infrastructure.2

Atlas (OpenAI, 2025) takes a similar approach with OpenAI’s model stack. The assistant can be invoked via a dedicated panel, and recent interactions are cached for short-term continuity. Like Comet, it is reactive, proprietary, and single-provider.

Both products represent meaningful steps toward AI-augmented browsing. However, they share three structural limitations: they are reactive rather than proactive, stateless across sessions, and closed-source with vendor-locked AI backends. ClawBrowser addresses all three.

3. Architecture

ClawBrowser is built on the host operating system’s native browser engine—WebKit on macOS, WebView2 on Windows, WebKitGTK on Linux—keeping the entire application under 10 MB. The AI agent runs as a persistent background process with deep hooks into the browsing layer. Four subsystems define its behavior.

3.1 Proactive Heartbeat

Rather than waiting for explicit commands, the agent maintains a continuous background pulse—a “heartbeat”—that monitors user activity across all open tabs. It observes browsing patterns, detects repetitive tasks, and surfaces contextual suggestions before the user asks. If the user has been comparing flights across three tabs for twenty minutes, the agent offers to consolidate the comparison. If a form resembles one filled out previously, it begins pre-filling fields.3

3.2 Persistent Memory

Context in ClawBrowser does not end with the session. The agent writes to and reads from a set of structured local files that encode long-term knowledge about the user:

SOUL.md captures personality, tone preferences, and communication style. USER.md stores factual preferences and biographical details—the user’s preferred airline, their manager’s name, their timezone. IDENTITY.md holds workflow patterns and role-specific context. These files are read on every launch and updated incrementally as the agent learns.

The effect is cumulative: the longer ClawBrowser is used, the less it needs to ask. It already knows the user’s defaults.

3.3 Self-Reflection Engine

Each night at midnight, the agent runs a self-reflection cycle.4 During this phase, it reviews the day’s interactions, extracts patterns, summarizes new knowledge, updates its memory files, and re-indexes its local state. By morning, the agent has consolidated what it learned and is incrementally better calibrated to the user’s behavior.

No other AI browser performs automated self-evaluation. The reflection engine is what transforms ClawBrowser from a tool with memory into a tool that learns.

3.4 Multi-Model Backend

ClawBrowser is not bound to a single AI provider. During setup, users select from OpenAI, Anthropic, Groq, local models via Ollama, or llama.cpp running entirely on-device. The system supports a two-tier configuration: a primary model for complex reasoning and a secondary model for fast, lightweight sub-tasks. Models can be swapped at any time without data loss or reconfiguration.

3.5 Local Voice Interface

Built-in speech recognition runs on-device using the operating system’s native speech framework. Audio never leaves the machine. Users can interact with the agent entirely by voice, entirely by keyboard, or through any combination of the two.

4. Privacy & Security

Every byte of user data—browsing history, AI conversations, memory files, API keys—is stored locally and encrypted with AES-256 behind a user-defined master password. Nothing is transmitted to any server unless explicitly directed by the user (e.g., when making an API call to a cloud-hosted model). The local-first architecture means that ClawBrowser functions fully offline when paired with a local model backend.

6. Availability

ClawBrowser is released under the MIT license. Source code, pre-built binaries for macOS, Windows, and Linux, and contribution guidelines are available at github.com/yungookim/ClawBrowser.

System requirements are minimal: any modern operating system, approximately 10 MB of disk space, and 8 GB of RAM if using local voice recognition or local AI models. Setup is handled through a guided wizard that completes in under sixty seconds.
